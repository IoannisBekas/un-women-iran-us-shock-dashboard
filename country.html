    Regional footprint

    Country points by exposure tier

    Map points use country centroids for orientation only. Colour follows exposure tier, size follows the 0-100 score and clicking a point selects that country.

    Calculated: point size is based on countries[].score; color is based on riskTier; coordinates are fixed country centroids.

    How to read: use the map for orientation and selection, not for subnational geography or affected-area mapping.

    Uses fields: latitude, longitude, score, riskTier.

    Selected-country gender and displacement lens

    Gender vulnerability, IPC, assistance and internal displacement are read together

    This panel keeps the gender proxy transparent: IPC is total population, internal displacement comes from the IDMC/IOM layer, and assistance pressure is included because assistance access and funding constraints can have differential gender effects.

    Calculated: the panel reports selected normalized model components, total-population IPC severity, IDMC/IOM displacement evidence and structural gender indicators.

    How to read: treat these as monitoring prompts for country teams and SADD follow-up, not measured women/girls food-security outcomes.

    Uses fields: genderProxy, components.*, indicators.*, genderFinance.*.

    Female labour and financial-access indicators

    These indicators are visible context for UN Women monitoring. They are structural/proxy evidence, not measured SADD food-security outcomes.

    Calculated: values are direct public-source indicators where available; no extra weight is applied in this table.

    How to read: use high vulnerability or low financial-access values to frame questions on coping capacity, access to assistance and livelihoods.

    Uses fields: informality, genderFinance.*, femaleHeadedHouseholdsPct.

    Supporting evidence

    What supplemental coverage adds

    Supplemental coverage counts five public-source context checks: ACLED conflict events, CPI, food incidents, PortWatch port activity and Findex gender-finance data. These layers explain data availability and context; they do not replace the published exposure score.

    Calculated: counts whether each supplemental source layer has public data for the selected country.

    How to read: coverage supports interpretation and identifies data gaps; it is not an additional risk score.

    Uses fields: supplementalEvidence.*, ACLED, CPI, food incidents, PortWatch and Findex coverage flags.

      Which pathway drives the selected country score

      0-1 components

      Bars are normalized 0-1 inputs before weights are applied. A longer bar means the selected country is high on that pressure pathway; missing inputs are set to a neutral value and flagged.

      Calculated: source indicators are transformed into bounded component_score values from 0 to 1 before index weights are applied.

      How to read: this chart shows pressure intensity by pathway; use the index decomposition next to see how weights convert those scores into final points.

      Uses fields: components.*, missingComponents, dataConfidence.

      Index decomposition

      How component scores become index points

      Weighted points

      This deconstructs the published Shock Exposure Index as component score x component weight x 100. It explains the score, not a causal pathway.

      Calculated: each row is component_score x component_weight x 100; contribution points sum to the published 0-100 Shock Exposure Index.

      How to read: the largest rows show which weighted components contribute most to this country's published score.

      Uses fields: components.*, indexWeights, score. See methodology.

      Readiness checks

      What the data can support now

      These are interpretation gates. They say whether the available public-source data can support a price event-study, food-security outcome model or gender-causal claim.

      Calculated: readiness flags are rule-based checks against source availability and whether required public fields exist.

      How to read: "not ready" means the public dataset cannot support that claim now; it does not mean the situation is low risk.

      Uses fields: readiness.*, dataConfidence, source-coverage flags.

      Supplemental coverage

      Public-data context available for this country

      These rows show whether useful public context exists for displacement, food-security outcomes and female-headed households. Missing coverage is a data gap, not evidence that the issue is absent.

      Calculated: each row is a source-availability check, not a severity calculation.

      How to read: available coverage means there is public context to inspect; unavailable coverage should be carried as a caveat.

      Uses fields: coverage, supplementalEvidence, readiness.
